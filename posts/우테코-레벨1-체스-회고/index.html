우아한테크코스 - 레벨1 체스 미션 & 레벨 1회고 (3월 22일 ~ 4월 8일)

By Seongwon Oh
Posted 2022-04-08 Updated 2022-05-08 14 min read

체스미션은 우테코의 꽃이라 불릴 정도로 유명한 미션 중 하나이다. 체스 미션은 이전 미션들에서 학습한 내용들을 모두 반영하고 4,5 단계에서 웹으로 적용까지 해보는 미션이었습니다. 미션을 진행하기 전까지는 이전 미션과 크게 다르지 않겠지!라는 생각을 갖고 있었지만 미션을 시작한 뒤에는 무언가 잘못 되었음을 느꼈습니다.

1, 2, 3 단계

1, 2, 3단계는 각각 체스판의 생성, 말들의 이동, 승패 판단의 기능들을 만드는 단계였습니다. 이전 미션과 같게 해당 미션은 페어 프로그래밍으로 미션을 진행했습니다. 이번 미션은 차리와 함께 진행을 하게 되었습니다.

객체 설계는 상태가 아닌 책임과 행동을 생각하자

미션을 진행하며 차리와 가장 많은 고민을 하였던 것은 도메인 설계에 대한 것이었습니다. 여러 종류의 말들과 체스 보드가 존재하는데, 과연 말들의 위치 정보를 누가 갖는 것이 더 좋은 설계일까?와 같은 고민을 하루종일 한 적도 있었습니다. 말들의 위치 정보를 보드가 가지면 체스 말들은 단순히 이동 연산만 하는 계산기의 역할만 하지 않을까? 그렇다고 체스 말이 위치 정보를 가지면 이동 경로 확인을 위해 너무 연산이 많아지지 않을까? 와 같은 고민을 했던 것 같습니다.

최근에는 객체지향의 사실과 오해의 책을 읽으며 상태를 중심으로 객체를 바라보는 것이 아니라 책임과 행동을 먼저 정한 후에 상태를 자연스럽게 정해야한다는 것을 알게 되어 고민들이 해소되었지만, 미션을 진행할 당시에는 도메인을 설계할 때 책임과 행동이 아닌 상태를 먼저 결정하려고 하여서 많은 고민을 했던 것 같습니다.

상태 패턴 & 체이닝

체스 미션을 진행하면서 1, 2, 3 단계를 진행할 때는 상태 패턴과 메서드 체이닝 기법을 사용했습니다.

상태 패턴의 경우는 지난 블랙잭 미션에서 네오가 강의를 해준 내용이자, 강의를 진행하면서도 상태 패턴의 경우 게임의 상태를 나타내기 위해 게임에 많이 사용된다는 이야기가 떠올라 사용을 했습니다. 미션에서 상태패턴을 사용하니 각각의 상태에 따라 사용 가능한 명령들과 사용 불가능한 명령들을 각각 다르게 적용할 수 있었습니다. 하지만 체스 미션 피드백 정리에 적은 내용과 같이 코드 내에 상태 패턴을 적용한 것에 대한 적합성, web연동시 발생하는 문제 때문에 4, 5단계에서는 제거를 하게 되었습니다.

체이닝은 체스 말 중 폰의 이동 로직을 구현할 때 사용했습니다. 해당 방법은 이전까지 접해보지 못한 새로운 방법이었습니다. 폰의 이동 로직은 초기 위치에서 두 칸 이동, 일반적인 한 칸 이동, 적을 공격하기 위한 대각선 이동과 같이 이동에 대한 로직이 복잡하기에 각각의 로직을 클래스로 생성하여 체이닝을 만들었습니다. 하지만 리뷰어 범블비의 피드백을 받고 생각을 해보니, 해당 이동 로직의 경우 순서가 중요하지 않아 체이닝의 사용은 적합하기보다 오버 프로그래밍에 가깝다는 생각이 들었습니다. 체이닝을 처음 사용해보아 코드는 남겨뒀으나, 제거를 하고 간단히 if문을 통해서도 충분히 구현할 수 있을 것 같습니다.

4, 5 단계

4 단계 - 웹 연결

4 단계는 콘솔로만 실행하던 게임을 웹 페이지를 간단하게 만들어서 게임을 하도록 구현하는 작업이었습니다. 해당 단계는 html, css, js의 경험이 많지 않았기에 기초 지식을 얻는데에 많은 시간을 투자했던 것 같습니다. 특히 js의 경우 완전히 무지한 분야이기에 이전 기수들의 미션 코드들을 보며 힌트를 얻고 동기/비동기 등의 학습 자료들을 찾고 학습을 하며 공부를 했습니다. 이렇게 기초 지식을 얻으며 학습을 하는데만 3일이 넘는 시간을 사용하였고 웹 작업을 겨우 마칠 수 있던 것 같습니다.

5 단계 - DB 연결

5 단계는 DB를 연동하여 게임을 이어서 할 수 있고, 여러 게임을 만들어 운영할 수 있도록 하는 단계였습니다. 여러 게임을 만들어 운영할 수 있는 기능은 추가적인 요구사항이었으나, 저는 4단계를 진행하며 이미 추가 요구사항의 구현도 마음을 먹었기에 망설임 없이 진행했습니다. 5단계의 경우, 제이슨과 포비의 강의를 모두 수강하여서 그런가 매우 쉽게 진행을 했던 것 같습니다. 다만 구현을 하며 앞선 1,2,3단계에서 구현을 하였던 상태패턴의 상태 관리를 하는 것에 어려움을 느껴 처음에는 Service가 상태를 인스턴스 변수로 갖는 잘못된 구현을 하기도 했었습니다. 이와 같이 서비스가 상태를 갖는 인스턴스 변수를 갖도록 구현을 하다보니 한번에 여러 게임이 실행될 수 없고 새로운 게임을 실행할 때마다 service를 초기화해줘야 하는 문제점이 있었습니다… 완전히 잘못된 설계였죠😭

해당 내용은 범블비의 피드백을 받으며 상태 패턴을 게임 내에서 삭제를 하는 결정을 하여 상태 패턴을 제거하고 service와 controller는 들어오는 request에 따라 DB의 값을 조회 및 로직 수행, 결과에 따른 DB값 업데이트를 하도록 변경했습니다.

그 외에도 여러 피드백을 받고 반영을 하며 우테코 레벨 1이 드디어 끝이 났습니다.

코로나 이슈..🤒

체스 미션을 시작하기 하루 전..그동안 잘 피해가던 코로나에 걸리게 되었습니다… 처음에는 목이 칼칼한 것으로부터 시작을 하여..시간이 가면 갈수록 몸살 기운이 생기기도하고 목이 많이 아프더군요😭 코로나 때문에 그동안 만나서 진행하던 페어 프로그래밍도 처음으로 각자 집에서 진행을하는 불편함을 경험했습니다..

저는 페어 기간동안 화,수,목,금,토까지 매일 6-9시간정도 페어와 게더를 통해 말을 하며 미션을 진행하다보니 목이 많이 아파져서 많이 힘들었어요😢 다들 코로나 조심했으면 좋겠어요..

글쓰기 미션

이번 주차에는 체스 미션뿐만 아니라 우테코 한 달 생활기라는 주제로 글쓰기 미션도 진행을 하였습니다. 글쓰기 미션의 경우, 프론트 크루들을 포함하여 6-7명 정도로 팀을 만들어 각자 글을 올리고 서로의 글을 읽고 피드백을 주는 형식으로 진행되었습니다.

글을 쓰고 다른 크루들의 글들을 읽어보니 평상시에 내가 느꼈던 생각, 감정들을 다른 크루들도 느끼고 있다는 것에 많은 공감이 되기도 하고 다른 크루들은 어떠한 학습법으로 어떤 우테코 생활들을 해왔는지 새롭게 알 수 있던것 같습니다.

개발뿐만 아니라 글쓰기 능력까지 책임져주는 우테코🔥 너무너무 좋아요

1단계 후기

우테코 1단계 과정을 진행하며 그동안 개발 공부를 하며 학습하지 못한 여러 공부를 하였던 것 같습니다. 대표적으로는 TDD와 Git, 클린코드 작성법, Java에 대한 학습 등이 있는 것 같습니다. 해당 내용들은 이전에 경험하지 못한 새로운 지식들이었기에 어려운 내용들이 있음에도 불구하고 흥미를 갖고 더욱 열심히 학습할 수 있던 것 같습니다. 특히 클린코드, 올바른 객체지향 코드 작성 등의 내용을 생각해보면 교육 시작으로부터 시간이 얼마 지나지 않았음에도 불구하고 자동차, 로또 미션 코드를 보면 부족함이 많이 느껴질 정도로 단기간동안 많은 발전을 한 것 같습니다. 다음 레벨때는 어떤 새로운 내용들을 학습할지 벌써부터 기대가 되네요😁 그리고 우테코를 진행하면서 개발 학습에 진심으로 임하는 여러 크루들을 만난 것 같습니다. 그들을 만나며 게더에서 여러 공부 이야기를 하며 제 학습의 부족함을 느끼기도 하고 더욱 더 열심히 공부를 해야겠다는 다짐을 하기도 했습니다.

이제 레벨 2가 시작되기 전까지 일주일이라는 방학 기간이 있습니다. 해당 기간때는 학습을 하며 놓친 부분과 깊게 공부를 하지 못한 부분들이 존재하는데 그러한 부분을 학습해봐야겠습니다. 그리고 그동안 시간이 없다는 핑계를 대며..미루고 있던 밀린 독서들도 해봐야겠습니다🔥
