# Spring Bean이란?

- Spring DI Container에서 관리되는 POJO(Plain Old Java Object)를 bean이라고 부른다.
  - POJO(Plain Old Java Object)로써 Spring 애플리케이션을 구성하는 핵심 객체이다.
  - Spring IoC 컨테이너(또는 DI 컨테이너)에 의해 생성 및 관리된다.
  - class, id, scope, constructor-arg 등을 주요 속성으로 지닌다.


## Spring Bean 구성 요소

- class: Bean으로 등록 될 Java클래스
- id: Bean의 고유 식별자
- scope: Bean을 생성하기 위한 방법 (singleton, prototype 등)
- constructor-arg: Bean 생성 시 생성자에 전달할 파라미터
- property: Bean 생성 시 setter에 전달할 인수

Spring에서는 bean들을 싱글톤으로 생성하여 관리를 한다.

## Bean 등록 방법

### @Component

- 스프링은 컴포넌트 스캔을 사용해 해당 어노테이션이 있는 클래스들을 찾아 자동으로 빈으로 등록해준다.
- 스프링은 기본적으로 컴포넌트 스캔을 이용해 빈을 등록하는 방법을 권장한다.
- 개발자의 입장에서도 해당 방법으로 빈으로 등록할 클래스에 @Component를 통해 빈을 등록하면 해당 클래스를 빈으로 등록되도록 잘 설정했는지 쉽게 파악할 수 있어서 좋다.

### @Configuration, @Bean

개발자가 직접 수동으로 스프링 컨테이너에 빈을 등록하기 위해서는 위의 두 annotation을 사용하면 된다.

```java
public class Resource { }

@Configuration
public class ResourceConfig {
	@Bean
	public Resource resource() {
			return new Resource();
	}
}
```

- Resource 클래스를 수동으로 빈으로 등록하려면 위와 같이 Config클래스를 생성하여 클래스에 `@Configuration`을 붙이고 Resource클래스를 빈으로 생성해주는 메서드에 `@Bean`을 붙여주면 된다.
  - 빈의 이름은 메서드의 이름으로 결정이 된다. (중복되지 않도록 주의할 것)
  - 스프링 빈으로 등록된 다른 클래스들에서도 `@Bean`을 통해 새로운 빈을 등록할 수 있지만, `@Configuraiton`이 붙은 클래스 안에서 `@Bean`을 통해 등록할 때만 싱글턴이 보장된다.
    - 만약 중복된 여러 빈을 생성하고 싶다면 `@Configuration(proxyBeanMetods= false)`를 붙여주면 된다.

    [[Spring] @Configuration 안에 @Bean을 사용해야 하는 이유, proxyBeanMethods - (2/2)](https://mangkyu.tistory.com/234)

- 동작 원리
  - 스프링 컨테이너는 기본적으로 `@Configuration`이 붙은 클래스를 빈으로 등록하고, 해당 클래스를 파싱하며 `@Bean`이 붙은 메서드를 찾아 빈들을 추가해준다.
- 수동으로 빈을 등록해줘야 하는 경우는 아래와 같다.
  - 개발자가 직접 제어가 불가능한 라이브러리를 사용할 때
    - Gson과 같이 가져다쓰는 클래스인 경우.
  - 애플리케이션 전범위적으로 사용되는 클래스를 등록할 때
  - 다형성을 활용하여 여러 구현체를 등록해줘야 할 때
    - `@Bean`을 통해 등록을 해주면 어떤 구현체들이 빈으로 등록되었는지를 `@Configuration`클래스만 보면 되어서 한 눈에 파악하기 쉽고 유지보수하기 좋다.

  → 가능하면 수동이 아닌 자동으로 등록하는 @Component방법을 사용하는 것이 좋다.
