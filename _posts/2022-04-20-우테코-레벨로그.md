# TDD란?
TDD는 Red, Green, Refactor의 순서로 진행을 하며 메인 로직을 작성하기 앞서 테스트 코드를 먼저 작성하는 순서로 진행하는 개발 방법입니다. 이러한 TDD는 디버깅 시간의 감소, 서비스의 안정성 증가, 개발자의 역량 강화 등의 이점이 있습니다.

# Package를 왜 사용할까?
패키지는 비슷한 성격의 자바 클래스들을 모아 놓은 자바의 디렉토리이다. 패키지는 관련된 클래스들을 효율적으로 관리하여 프로젝트나 소프트웨어간의 소스코드의 충돌을 방지해주고 소스코드를 묶어 배포와 재사용을 할 수 있게 해준다.

#클래스 필드와 인스턴스 필드의 차이점 및 각각 사용하는 이유
- 클래스 필드와 인스턴스 필드의 차이점은 외형적으로는 static이 붙은 유무가 있으며 사용 측면에서 보면 클래스 필드는 미리 선언을 하지 않고 사용할 수 있고 인스턴스 필드는 무조건 해당 객체를 만들고 사용해야 한다.
- 인스턴스 필드는 서로 다른 상태를 가져야 할 때 사용하며 클래스 필드는 공유하는 정보가 있을 때 사용한다.

#String+String과 StringBuilder의 차이
String은 불변하기에 String 두 개를 합치면 더할 때마다 새로운 객체를 생성한다. Java8 이후로는 내부적으로 컴파일시 최적화를 해주지만 해주지 않는 경우도 존재합니다.
반면에 StringBuilder은 기존 데이터에 새로운 데이터를 더하는 방식을 사용해 메모리와 속도 측면에서 더 빨라서 긴 문자열을 더할 때는 StringBuilder를 사용하여 낭비를 줄여야 한다.

# 동일성과 동등성?
동일성(==)은 객체가 참조하고 있는 주소 값을 비교하는 것이며 동등성(equals)는 equals를 통해 정의된 값에 따라 비교를 하는 것이다.

#Generic의 장점
제너릭은 클래스나 메서드에서 사용할 내부 데이터 타입을 컴파일시에 미리 지정하는 방법이다. 제너릭의 장점은 특정 타입을 제한함으로써 타입 안정성을 제공하고 외부에서 타입을 지정해주었기에 타입을 따로 체크하고 변환할 필요가 없어 관리가 편리하다.

# Checked Exception과 Unchecked Exception의 차이
Checked는 Exception을 상속하여 컴파일 시점에 예외를 catch하는지 확인하여 EXCEPTION이 발생하는 메서드에서는 throws를 해줘야 한다. 반면에 Unchecked는 RuntimeException을 상속하며 컴파일 시점에 예외를 catch하는지 확인하지 않아 throws예약어를 활용해 예외를 처리하지 않아도 된다.

# 추상화란?
추상화는 내부의 동작(how)를 감추고 무엇(what)을 하는지만 드러내어 사용자들이 구현이 아닌 인터페이스에 의존하도록 하는 방법이다. 추상화를 사용하면 코드의 재사용성 증가, 가독성 향상, 생산성 증가로 유지보수 향상의 장점이 있습니다.

# 추상 클래스와 인터페이스의 차이
인터페이스는 100%로 추상화를 하며 포멧만 제공하여 필드를 소유할 수 없다. 하지만 추상 클래스는 인스턴스 필드를 소유할 수 있다. 인터페이스는 클라이언트 쪽에서 사용하도록 만드는 것이 좋고, 추상 클래스는 내부에서 기능을 확장할 때 쓰는 것이 좋다.

# 상속과 조합의 차이
상속은 extends를 통해 기존에 정의되어 있는 클래스의 필드와 메서드를 물려받아 새로운 클래스를 생성하는 기법이며, 조합은 기존 클래스를 확장하는 대시 새로운 클래스에 private필드로 기존 클래스의 인스턴스를 참조하게 하는 방법이다.

# 상속의 단점
상속은 하위 클래스가 상위 클래스의 구현에 의존하여 변경에 취약하다. (결합도가 높아진다.)
상위 클래스의 모든 퍼블릭 메서드가 하위 클래스에 노출되어 불필요한 메서드들도 상속받는 문제가 발생할 수 있다.

# 상속과 조합의 사용
상속의 경우 단점이 많기에 상속을 고려할 때는 Is-a관계인지, 클라이언트 관점에서 두 객체가 동일한 행동을 할 것이라 기대하는지 고려해보고 사용해야한다. 그 외에 단순하게 코드를 재사용하며 중복 코드를 줄이고 싶다면 상속보다는 조합을 고려해야 한다.
